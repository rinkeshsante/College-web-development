Instructions to use system:

    this project works on the heierarchy order as:
  1. admin
  2. sub_admin
  3. department members
  4. new users / outsiders

After creating new account user needs aproval of sub_admin to get read access from sub_admin.project can have many sub admin. optionally he can create issue to grab attention of sub admin. after aproval from sub admin he will be departmental members like teachers , students (optionally) or lab assistant.

Then sub admin can view all data availble on website and dowload .csv reports from tables of various entites. he can view in depth data and view relationships between entites.

Any chages like create , update and delete can be only done by sub_admin and admin. admin has more option than sub admin. In case of department member want to suggest any chages in db he/she can create public issue and actual sub_admin can make chages according to issue and clear issue after that. its resposiblity of sub_admin to keep minimum entries in issue list.
